概述
----------------------------------------------------------------

NxS是一个NSIS插件，用于显示带有进度条和取消按钮的横幅。
===================================================================
用法
插件入口点：Show、Update、Destroy、HasUserAborted、getWindow

nxs::Show /NOUNLOAD "对话框标题文本" \
  /top "顶部文本" \
  /sub "子文本（更多信息）" \
  /h 1 \
  /pos 43 \
  /max 100 \
  /can 1 \
  /end
提示：将以上内容复制粘贴到您的脚本中，并修改参数。
-----------------------------------------------------------
参数说明：
-------------------------------------------------------------
/top：设置顶部文本（在图标右侧），最多3行。
/sub：设置子文本（进度条控件上方），最多3行。
/h：显示/隐藏任务栏对话框按钮。需要一个数字：1表示隐藏，0表示显示。
/pos：设置进度条位置。
/max：设置进度条控件的步数。默认值为100。
/can：启用/禁用对话框上的取消按钮。需要一个数字：1表示启用，0表示禁用。默认禁用。
/marquee：设置跑马灯样式和进度条移动速度。0表示取消跑马灯样式，大于0的值表示移动间隔（以毫秒为单位），建议使用50或100。
/end：停止堆栈读取。
==============================================
注意：
===================================================
/pos参数的值范围为0（零）到100，或者是/max参数指定的值。
堆栈上传递的其他文本（不是参数之一）将用作对话框的标题。如果传递多个未被识别为参数的字符串，则最后一个字符串将用作标题，其余的字符串将被忽略。
Update命令支持Show命令的任何参数，但在大多数情况下，使用更简单：

nxs::Update /NOUNLOAD /top "正在检查..." /sub "78% 完成" /pos 78 /end
要获取对话框的句柄（HWND），请使用：

nxs::getWindow /NOUNLOAD
   Pop $0 ; $0现在包含对话框窗口的句柄
提示：您无需使用getWindow来修改文本。只需使用以下命令：

nxs::Update /NOUNLOAD "新对话框标题（标题）"
要检查取消按钮是否被按下，请使用：

nxs::HasUserAborted /NOUNLOAD
    Pop $R0 ; $R0现在包含"1"（如果用户点击了取消按钮）或 "0"（零）（如果用户未点击）。
要销毁对话框并释放资源，请使用：

nxs::Destroy
请参考Example.nsi文件中的示例。
